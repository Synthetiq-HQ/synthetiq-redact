# Public Open Source Release Checklist

Before public release:

- No real uploads, OCR outputs, processed files, local databases, screenshots, or reports are committed.
- Fresh clone setup works.
- Synthetic demo data only.
- README states limitations and does not overclaim compliance.
- Security policy is present.
- Contribution guide is present.
- Responsible-use policy is present.
- Licence is selected.
- Third-party dependency and model licence matrix is complete.
- Backend tests pass.
- Frontend build passes.
- Frontend dependency audit passes.
- Secret scan is run.
- Redaction benchmark report is generated.
- Handwriting limitations are documented.
- Release notes include known risks.
- Public issue templates separate bugs, model misses, security reports, and feature requests.
